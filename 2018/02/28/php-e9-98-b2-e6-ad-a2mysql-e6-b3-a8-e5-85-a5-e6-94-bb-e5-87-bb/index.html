PHP防止mysql注入攻击

很多web平台在接受参数时，为进行严格的参数验证，导致数据库接收到一堆垃圾数据，甚至数据库访问权限被恶意访问者获取。 示例参考http://php.net/manual/zh/security.database.sql-injection.php 防范这些sql注入的基本流程。 1.输入参数的验证 类型验证或者进行强制的类型设置。 isset — 检测变量是否已设置并且非 NULL empty — 检查一个变量是否为空 is_numeric — 检测变量是否为数字或数字字符串 ctype_digit — 做纯数字检测 settype — 设置变量的类型 sprintf — Return a formatted string 长度验证 长度验证需要配合类型验证同步进行。 strlen 正则验证 2.参数化的sql查询，使用sprint进行参数格式化处理或者mysql_real_escape_string函数进行参数处理。 例如 settype($offset, 'integer'); $query = "SELECT id, name FROM products ORDER BY name LIMIT 20 OFFSET $offset;"; // 请注意格式字符串中的 %d，如果用 %s 就毫无意义了 $query = sprintf("SELECT id, name FROM products ORDER BY name LIMIT 20 OFFSET %d;", $offset); 或 mysql_real_escape_string函数PHP7已不支持可以使用mysqli_real_escape_string // Connect $link = mysql_connect('mysql_host', 'mysql_user', 'mysql_password') OR die(mysql_error()); // Query $query = sprintf("SELECT * FROM users WHERE user='%s' AND password='%s'", mysql_real_escape_string($user), mysql_real_escape_string($password));
