CGI和FAST-CGI

cgi是web server保证web server传递过来的数据是标准格式的协议。

当web服务器接收到请求，会启动对应的CGI程序，比如php的解释器，php解释器会解析php.ini，初始化执行环境，然后处理请求，再按照CGI规定的格式返回处理后的结果，退出进程，web服务器将数据传递给浏览器。

fast-cgi是用来提高cgi性能的

因为每一个请求都会需要执行php的解释器，php解释器会解析php.ini，初始化执行环境。执行完成后，杀掉这些进程。

fast-cgi会先启动一个master，解析配置文件，初始化执行环境，在启动多个worker，请求来的时候会传递给worker，然后可以接受下一个请求，因为master会启动多个worker进程去处理请求，就提高呢性能，同时worker也会自动在空闲时停掉。

fast-cgi 为nginx的一个模块

php-fpm

php-fpm是实现了fast-cgi的程序，主要用来调度php-cgi的进程。

fast-cgi 调用php-fpm->php-cgi

执行流程

www.example.com
|
|
Nginx
|
|
路由到www.example.com/index.php
|
|
加载nginx的fast-cgi模块
|

fast-cgi监听127.0.0.1:9000地址

|

|
|

|
|
nginx将结果通过http返回给浏览器
